웹 표준(Web standards)

웹 표준이란 웹에서 표준적으로 사용되는 기술이나 규칙을 말합니다.

웹 표준이 등장하게 된 배경

1. 인터넷 초기 : 인터넷 익스플로러(Internet Explorer)를 이용해 인터넷에 접속

2. 이후 : 여러 브라우저의 등장 (크롬, 오페라, 엣지, 사파리 등)

3. 문제점 발생 : 익스플로러를 제외한 다른 브라우저에서 다르게 보여지거나 멀티미디어 기능이 제대로 작동하지 않는 오류 발생

4. 대안 작업 : W3C(World Wide Web Consortium)를 중심으로 웹 페이지의 표준화에 대한 작업이 진행

웹 표준안은 브라우저 중심이 아닌 HTML 태그가 중심이고, 사용자 중심이 되는 표준안입니다.

따라서 모든 브라우저에서 공통으로 인터넷을 사용하여 자유롭게 커뮤니티를 만들어가야하기 때문에, 웹 표준이 필요하게 되었습니다.

W3C(World Wide Web Consortium)

표준화 단체인 W3C가 권고한 표준안에 따라 웹사이트를 작성할 때 이용하는 HTML, CSS, JavaScript 등에 대한 규정이 만들어졌습니다.

즉, 어떤 운영체제나 브라우저를 사용하더라도 웹페이지가 동일하게 보이고 정상 작동해야함을 의미합니다.

현재 HTML5로 문서를 만들고 그에 따른 웹 표준을 준수하는 것이 기본이라고 할 수 있습니다.

컴퓨터가 읽고 표현할 수 있는 언어인 HTML과 CSS를 문법에 맞추어 사용한다.

표시가 없을 때 보다 표시가 있을 때, 보기가 편하고 어떤것이 제목인지, 목록인지, 내용인지 등 구분이 되어있습니다.
사람들은 위의 컨텐츠를 보고 내용 파악이나 문단 구분이 가능할 수 있지만, 컴퓨터는 구분이 불가능합니다.

제작자가 의도한 방향대로 정확하게 문서를 표현하는 것이 중요하며, 이 모든 것은 마크업 언어를 통해 구조적으로 표현이 가능합니다.

따라서, 올바른 문법, 올마른 마크업 언어를 사용하는 것이 퍼블리싱에서 매우 중요합니다.

마크업 언어 사용 목적

웹브라우저와 사람들에게 정확한 정보를 제공하기 위해서 마크업 언어를 사용합니다. 웹 브라우저에게 태그를 사용하여 명령을 하고, 동일한 언어(태그)로 표현하기 때문에 작업자들이 이해하고 수정, 보완하기에 편리합니다.

또한 정보를 수집하는 웹크롤러라는 로봇이 HTML 문서 내에 있는 태그를 읽고 그 태그를 통해 구조를 파악할 수 있으며, 오류 없이 정보를 이해하고 전달하여 사용자들이 검색하기 수월하게 도와줍니다.

즉, 마크업 언어는 웹 브라우저에게 컨텐츠를 어떻게 표현해야 하는지 명령하는 언어이며, 함께 일하는 사람들뿐만 아니라 웹 크롤러라는 데이터를 긁어 수집하는 로봇들에게도 웹 문서 내의 데이터들을 구조적으로 표시하여 정보를 전달하는 언어입니다. 이것이 HTML에서의 마크업 언어 입니다.
