시맨틱 구조

웹 표준 문서 HTML5의 구조가 복잡해짐에 따라 소스 코드에서 구조를 파악하기 쉽지 않아 시맨틱 태그(Semantic tag)를 사용하도록 권장하고 있습니다.

시맨틱 태그는 HTML5에서 새롭게 추가된 태그로 페이지의 구조를 쉽게 파악할 수 있게 도와줍니다. 또한 검색 엔진 같은 프로그램에서 자료를 검색하여 정보의 의미를 분석할 수 있게 도와주는 지능형 웹이며, 시각 장애인들을 위한 웹 표준 방식이기도 합니다.

시맨틱 태그의 필요성

시맨틱 태그가 나오기 전 웹 페이지는 구조를 만들기 위해 대부분 코드에 <div> 태그를 사용하고 id 속성이나 class 속성으로 구분했습니다. 이 때, id 나 class의 속성은 대부분 작업자에 의해 지정되었습니다.

그래서 코드를 수정하거나 구조를 파악하기 위해서 많은 어려움이 있었고, 시각 장애인이 화면 낭독기(스크린 리더기)를 통해 검색 엔진에서 웹 소스를 읽을 때 어느 부분이 메뉴이고 본문인지 파악하기가 어려웠습니다.

이러한 문제를 해결하기 위해 구조에 들어가는 태그를 미리 약속하고 해당 태그를 지정한다면 이해하기 쉬운 구조가 될 것입니다. 그래서 HTML5 표준안에서 문서 구조를 만드는 시맨틱 태그가 등장했습니다.

시맨틱 태그는 페이지 구조에서 어떤 역할을 하는지 이름으로 쉽게 이해할 수 있습니다. 예를 들어 <nav> 태그는 navigation 의 성격을 갖고 있는 태그입니다.

그렇다면 우리가 흔히 보는 웹 사이트에서 시맨틱 태그가 어떤 것이 있는지, 어떻게 활용되고 있는지 알아볼까요?

시맨틱 태그의 종류

1) <header> 태그

페이지 맨 위쪽에 삽입하는 머리말을 의미합니다. 사이트 전체의 헤더는 주로 로고를 중심으로 내비게이션 메뉴와 검색 창이 들어가며, 내비게이션 메뉴는 <nav> 태그로 구성할 수 있습니다.

2) <nav> 태그

nav 태그의 사용으로 어느 부분이 메뉴인지 쉽게 파악할 수 있으며, 사이트 내의 내비게이션 메뉴를 묶거나 독립적으로 구성할 수 있습니다. 문서 안에 여러 개의 nav 태그를 사용할 경우, id 선택자로 구분하여 스타일을 꾸미고, 구분하여 독립적으로 사용하도록 권장합니다.

3) <section> 태그

콘텐츠 영역을 포함하는 태그로 해당 콘텐츠를 주제별로 묶을 때 사용하며 <section> 태그 내부에는 h1~h6의 제목 태그가 함께 사용됩니다. 또한 다른 태그(div, span, img, a... 등)를 넣어 사용합니다.

4) <article> 태그

article의 사전적 의미는 신문이나 잡지의 ‘기사’를 뜻하는 것으로 <article> 태그는 웹 상의 실제 내용을 작성하는 것입니다. 블로그의 포스트나 웹 사이트 내용과 같은 독립적인 웹 콘텐츠가 이에 해당됩니다.

5) <aside> 태그

웹 문서의 왼쪽이나 오른쪽 또는 하단에 사이드 바가 표시되는 것을 볼 수 있는데, 이런 사이드 바를 만드는 태그가 aside 태그 입니다. aside 태그는 필수 요소가 아니며 광고나 일반적인 링크 모음처럼 문서의 메인 내용에 영향을 미치지 않는 내용을 넣을 때 사용합니다.

6) <footer> 태그

웹 문서 하단의 저작권 표시와 같은 내용과 연락처 정보를 표시합니다.
